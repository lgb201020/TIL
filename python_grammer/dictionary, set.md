# 사전 자료형과 집합 자료형
## 사전 자료형
- 키(key)와 값(value)의 쌍을 데이터로 가지는 자료형으로 리스트나 튜플이 순차적으로 데이터를 저장하는 것과는 대비된다.
- 사전 자료형은 키(key)와 값(value)의 쌍을 데이터로 가지며 Immutable(변경 불가) 자료형을 키로 사용할 수 있다.
- 파이썬의 사전 자료형은 <U> ***해시 테이블(Hash Table)*** 을 이용하므로 데이터의 조회 및 수정에 있어 복잡도 O(1)의 시간에 처리할 수 있다.</U>

    ```
    # 사전 자료형 초기화 방법1
    data = dict()
    data['사과'] = 'apple'
    data['바나나'] = 'banana'
    data['코코넛'] = 'coconut'

    print(data) # {'사과': 'apple', '바나나': 'banana', '코코넛': 'coconut'}

    # 사전 자료형 초기화 방법2
    data = {'사과': 'apple', '바나나': 'banana', '코코넛': 'coconut'}
    ```

- 사전 자료형에서는 key와 value를 별도로 뽑아내기 위한 메서드를 지원한다.
  - key(): key 데이터만 뽑아서 dict_keys객체로 반환하는 메서드
  - value(): value 데이터만 뽑아서 dict_values 객체로 반환하는 메서드
    ```
    key_list = data.key()
    value_list = data.value()

    print(key_list) # dict_keys(['사과', '바나나', '코코넛'])
    print(value_list) # dict_values(['apple', 'banana', 'coconut'])
    ```

## 집합 자료형
- 집합은 순서가 없고 중복을 허용하지 않는다.
- 집합은 리스트 혹은 문자열을 활용해 초기화 할 수 있고 이때 set() 함수를 사용한다.
- 집함은 중괄호({}) 안에 각 원소를 콤마(,)를 기준으로 구분하여 삽입함으로써 초기화 할 수 있다.
- <U>데이터의 조회 및 수정에 있어 복잡도가 O(1)이다.</U>
- 집합은 데이터 존재 여부를 판단할 때 매우 효과적으로 활용 가능하다.
    ```
    # 집합 자료형 초기화 방법 1
    data = set([1, 1, 2, 3, 4, 4, 5])
    print(data) # {1, 2, 3, 4, 5}가 출력

    # 집합 자료형 초기화 방법 2
    data = {1, 1, 2, 3, 4, 4, 5}
    print(data) # {1, 2, 3, 4, 5}가 출력
    ```
## 집합 자료형의 연산
- 집합 자료형은 기본적으로 수학에서 제공하는 집합 연산을 제공한다.
  - 합집합: 집합 A에 속하거나 B에 속하는 원소로 이루어진 집합으로 OR 연산
  - 교집합: 집합 A와 집합 B에 동시에 속하는 원소로 이루어진 집합으로 AND 연산
  - 차집합: 집합 A에 속하는 원소 중에서 B에 속하지 않는 원소들로 이루어진 집합 뺄셈 연산
    ```
    a = set([1,2,3,4,5])
    b = set([3,4,5,6,7])

    # 합집합
    print(a|b)
    
    # 교집합
    print(a&b)

    # 차집합
    print(a - b)
    ```

## 집합 자료형 관련 함수
```
a = set([1,2,3,4,5])

# 원소 추가
a.add(6)

# 원소 여러 개 추가
a.update([7,8,9])

# 특정 값을 갖는 원소 삭제
a.remove(3)
```

## 사전 자료형과 집합 자료형의 공통점
- 리스트나 튜플은 순서가 있어 인덱싱을 통해 자료형의 값을 얻을 수 있으나 사전 자료형이나 집합 자료형은 순서가 없어 인덱싱으로 값을 얻을 수 없다.
- 사전의 키 혹은 집합의 원소를 이용해 O(1)의 시간 복잡도로 값 조회가 가능하다.
