# 알고리즘 성능 평가
## 복잡도(complexity)
- 복잡도는 알고리즘의 성능 평가 척도이다.
  - 시간 복잡도: 특정 크기 입력에 대한 알고리즘 수행 시간 분석
  - 공간 복잡도: 특정 크기 입력에 대한 알고리즘 메모리 사용량 분석
- 동일한 기능 수행 알고리즘이라면 일반적으로 복잡도가 낮을수록 좋은 알고리즘이다.

## 빅오표기법(Big-O notation)
- 가장 빠르게 증가하는 항만을 고려하는 표기법으로 계수는 무시한다.
- 예를 들어 연산 횟수가 $3N^3 + 5N^2 + 1,000,000$인 경우 빅오 표기법에서는 가장 차수가 큰 항만 남기고 계수는 무시하므로 $O(N^3)$로 표현된다.

## 시간 복잡도 계산
- N개의 데이터 합을 계산하는 경우
  
    ```
    array = [3, 5, 1, 2, 4]
    summary = 0

    for i in array:
        summary += x

    print(summary)    
    ```
    - 수행 시간은 데이터의 개수가 N에 비례할 것임을 예측할 수 있다.
    - 시간 복잡도: $O(N)$
- 2중 반복문을 이용하는 프로그램 예제

    ```
    array = [3, 5, 1, 2, 4]

    for i in array:
        for j in array:
            temp = i * j
            print(temp)
    ```
    - 시간 복잡도: $O(N^2)$
    - 모든 2중 반복문의 시간 복잡도가 $O(N^2)$인 것은 아니며 소스코드 내부적으로 다른 함수를 호출한다면 해당 함수의 시간까지 시간 복잡도로 고려해야 한다.

## 알고리즘 설계 팁
- 일반적으로 CPU 기반 컴퓨터에서 연산 횟수가 5억이 넘어가는 경우:
  - C언어 기준 보통 1~3초 가량의 시간이 소요됨
  - python 기준 보통 5~15초 가량의 시간이 소요됨
- 코딩 테스트 문제에서는 시간제한이 통상 1~5초 가량이라는 점을 유의해야 하며 문제에 시간 제안이 없는 경우 시간제한이 대략 5초 정도라고 생각하고 문제를 푸는 것이 합리적이다.
- 문제에서 가장 먼저 확인해야 하는 내용은 조건 및 제한이고 그 중 시간제한을 가장 먼저 생각해야 한다.
- 시간제한이 1초인 경우 일반적인 기준은 다음과 같음
  - N의 범위가 500인 경우: $O(N^3)$
  - N의 범위가 2000인 경우: $O(N^2)$
  - N의 범위가 100,000인 경우: $O(NlogN)$
  - N의 범위가 10,000,000인 경우: $O(N)$

## 일반적인 알고리즘 문제 해결 과정
- 일반적인 알고리즘 문제 해결 과정은 다음과 같다.
  
  1. 지문을 읽고 컴퓨터적 사고를 통해서 문제를 작은 단위들로 분해해 정리
  2. 요구사항을 보고 조건과 제한을 고려
  3. 문제 해결을 위한 핵심 아이디어를 탐색
  4. 소스코드 설계 및 코딩
- 일반적으로 대부분의 문제는 핵심 아이디어를 캐치한다면 간결하게 소스코드를 작성해 문제를 해결할 수 있는 형태로 출제된다.
- 수행시간 측정
  ```
  import time
  strat_time = time.time()
  
  # 프로그램 소스코드
  
  end_time = time.time()
  print("time:", end_time - strat_time) # 코드 수행 시간 출력
  ```
